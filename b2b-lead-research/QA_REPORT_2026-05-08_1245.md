### ❌ Failed Tests (Root Cause Analysis)

#### 1. `tests/test_enrichment.py::TestWebsiteEnricher::test_analyze_design`
```
assert result["has_schema_markup"] is True
assert False is True
```
- **Root Cause:** The test passes `html = "display: flex;"` to `_analyze_design()`, but expects `has_schema_markup=True`. The schema markup (`application/ld+json`) is only in the `soup` object, not in the `html` parameter.
- **Fix:** Update test to include `"application/ld+json"` or `"schema.org"` in the `html` string passed to `_analyze_design()`.
- **Severity:** **WARNING** (test bug, not code bug)

#### 2. `tests/test_pipeline.py::TestLeadPipeline::test_score_leads_handles_errors`
```
assert result[0]["website_quality_score"] == 50
assert 38.0 == 50
```
- **Root Cause:** The test expects a default score of 50 for an empty lead, but `ScoringEngine.calculate()` produces 38.0 for a lead with all `None` fields. No exception is thrown, so the fallback `score=50` in the error handler is never reached.
- **Fix:** Either change test expectation to 38, or mock `scorer.calculate()` to raise an exception to actually test error handling.
- **Severity:** **WARNING** (test bug)

#### 3. `tests/test_scoring.py::TestScoringEngine::test_calculate_full`
```
assert score == 100
assert 95 == 100
```
- **Root Cause:** Test uses `page_speed_score=85`, which scores 10 points (threshold for 15 points is ≥90). All other metrics sum to 85, giving total 95.
- **Fix:** Change `page_speed_score` to 95 (or any ≥90) to achieve perfect 100, or update assertion to 95.
- **Severity:** **WARNING** (test bug)

#### 4. `tests/test_ai_copywriter.py::TestAICopywriter::test_call_llm_api_error` *(NEW)*
```
Exception: Connection timeout
```
- **Root Cause:** Test mocks `requests.post` with `side_effect = Exception("Connection timeout")`, but the code only catches `requests.RequestException` and JSON parsing errors. A generic `Exception` is not caught, so it bubbles up uncaught.
- **Fix:** Change test to use `requests.RequestException("Connection timeout")` or `requests.ConnectionError("Connection timeout")`.
- **Severity:** **WARNING** (test bug)

#### 5. `tests/test_webhook.py::TestWebhookExporter::test_send_generic_failure` *(NEW)*
```
Exception: Timeout
```
- **Root Cause:** Same pattern as above — test uses `mock_post.side_effect = Exception("Timeout")` but `_send_generic()` only catches `requests.RequestException`.
- **Fix:** Change test to use `requests.RequestException("Timeout")` or a subclass.
- **Severity:** **WARNING** (test bug)
